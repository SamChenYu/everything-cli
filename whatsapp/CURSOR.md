# WhatsApp CLI - Cursor Context

## Overview
Node.js Playwright CLI tool for WhatsApp Web (personal use). Uses Ink (React) for the terminal UI and Playwright to automate a Chromium instance pointed at web.whatsapp.com.

## Architecture
- `src/index.tsx` — entry point, renders the Ink app
- `src/cli/app.tsx` — Ink React UI (chat selection, message viewing, sending)
- `src/whatsapp/WhatsAppService.ts` — Playwright automation (browser launch, DOM scraping, message sending)

## WhatsApp Web Selectors
All DOM selectors are configurable via `.env` (see `.env.sample`). This is critical because WhatsApp Web uses obfuscated/minified CSS class names that change on every deploy.

### Selector stability tiers
- **Stable:** `data-testid` attributes (e.g. `selectable-text`, `qrcode`, `conversation-compose-box-input`), `role` attributes, `aria-label`
- **Unstable:** Any CSS class name starting with `_` or `x` (e.g. `._ahxt`, `.x1c4vz4f`) — these are generated by WhatsApp's CSS bundler and break frequently

### Key design decision: `data-pre-plain-text`
Messages are scraped via the `data-pre-plain-text` attribute on message elements, which contains sender name, time, and date in the format `[2:04 pm, 16/03/2026] SenderName: `. This replaced the old approach of targeting individual child elements by obfuscated class names for timestamp and sender. The `parsePrePlainText()` function extracts all metadata from this single attribute.

### Known broken selectors (as of March 2026)
- `[data-testid="conversation-panel-messages"]` — no longer exists. Removed; we now wait for `[data-pre-plain-text]` elements directly.
- `.message-out` — used for outgoing message detection, may be obfuscated. Has a fallback: checks `data-id` attribute prefix (`true_` = outgoing).

## Gotchas
- Media messages (images, stickers, voice notes) have `data-pre-plain-text` but no `span[data-testid="selectable-text"]` inside. The `textContent()` call uses a 1-second timeout to fail fast and show "(media)".
- After clicking a chat, messages may not render immediately. `openChat()` captures the first message's metadata before clicking, then waits for it to change (detects panel refresh) before reading new messages.
- Session is persisted via `launchPersistentContext` using a full Chrome profile directory (`.whatsapp-chrome-data/`). This preserves IndexedDB and Service Workers — `storageState` (cookies + localStorage only) was insufficient for WhatsApp Web auth.
- Message sending uses `document.execCommand('insertText')` instead of Playwright's `fill()` or `pressSequentially()`. WhatsApp's contenteditable compose box requires the `beforeinput`/`input` event chain that only `execCommand` triggers properly. The send button (`[data-testid="send"]`) is clicked as the primary submit method, with Enter as fallback.
- Run with `npm start` (uses `tsx src/index.tsx`). Requires `npm install` first.
